## Evoxt 备份的第一个惊喜：每周自动异地备份，不要钱

很多 VPS 厂商的备份功能要么收费，要么只是快照（存在同一台物理机上，物理机挂了你的"备份"也没了）。

Evoxt 的做法是：**所有套餐默认包含每周自动异地备份，零额外费用**。

当然这只是强调安全性，但侧面说明了备份的独立程度。

## 免费备份够用吗？什么情况下需要升级付费方案

每周一次的备份，对个人博客、学习用机、低流量小项目来说已经够了。

但如果你跑的是：

- **电商网站**：每天都有新订单、新用户数据
- **数据库应用**：内容更新频率高
- **自动化脚本 / 爬虫**：数据实时在变
- **生产环境服务**：数据丢一周等于损失巨大

那一周一次的备份窗口可能太宽了。一周内任何一天出问题，最多可能要回滚 7 天前的状态。

### 付费备份方案提供什么

Evoxt 的付费备份计划（可在 VM 控制面板 → Upgrade 标签里开通）主要升级了两点：

1. **每日备份**：从每周一次升级为每天自动备份，数据丢失窗口从最多 7 天压缩到最多 1 天
2. **5 份轮转备份**：保留 5 个历史还原点（免费版只有 1 份），可以选择恢复到更早的某个时间点
3. **手动备份触发**：需要时可以临时手动触发一次备份，比如在做大版本升级之前

付费备份的定价根据 VM 存储容量变动，在 VM 控制面板里查看对应计划的具体报价。

**建议：** 如果你的业务涉及真实用户数据或交易记录，付费的每日备份 + 5 份轮转是很值得的保险成本。

## 除了备份，Evoxt 还附带哪些功能

聊了这么多备份，顺便把 Evoxt 其他常用功能也梳理一下，方便你评估这是不是适合自己的 VPS：

**克隆（Clone）**：把一台 VM 整个复制一份，用于测试环境、灾难预案或多地域部署。省去重新配置的时间。

**防火墙（Firewall）**：在控制面板里直接设规则，不用 SSH 进服务器操作。Layer 3 级别防护，能挡住一定强度的 DDoS 流量。

**VNC 访问**：服务器 SSH 挂了、系统启动失败，用 VNC 直接连控制台救机，不用提交工单等人工协助。

**IP 地址管理**：支持浮动 IP（Floating IP），可以在多台 VM 之间切换绑定，用于做故障转移或高可用架构。

**一键应用部署**：WordPress（带 OpenLiteSpeed）、LAMP/LEMP、Docker、GitLab、Nextcloud 等常见应用，部署页面直接选，省去手动装环境的步骤。

**Windows VPS**：支持 Windows Server + RDP 访问，不额外收授权费，价格和 Linux 套餐一样。
